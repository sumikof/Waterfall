- 性能テスト目標値を達成
- セキュリティ診断の指摘事項を対処済み

**受入テスト（UAT）:**
- 顧客側テスト合格および承認

## 7. 不具合管理方針

### 7.1 重大度定義

| 重大度 | 定義 |
|---|---|
| S1 (Critical) | サービス停止・データ損失レベル |
| S2 (Major) | 主要機能が使用不可 |
| S3 (Minor) | 機能は使用できるが動作不正 |
| S4 (Trivial) | 軽微な表示・文言の問題 |

### 7.2 対応方針
- S1: 即時対応（開発継続停止）
- S2: スプリント内対応
- S3/S4: 次スプリント以降で対応
